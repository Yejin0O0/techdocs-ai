# PRD — TechDocs AI

## 배경 및 문제 정의

팀 내 기술 문서가 누적될수록 필요한 정보를 찾는 데 점점 더 많은 시간이 걸린다. 실무자는 모르는 것이 생길 때마다 동료에게 직접 물어보거나 수십 개의 문서를 일일이 뒤져야 하고, 이는 팀 전체의 생산성을 떨어뜨린다.

기존 해결책(사내 위키, 공유 드라이브, Notion)은 문서를 저장하지만 **검색 의도를 이해하지 못한다.** 키워드 검색은 정확한 단어를 알아야 하고, 출처가 없는 답변은 신뢰하기 어렵다.

## 솔루션

기술 문서를 업로드하면 AI가 즉시 검색하고 **출처와 함께** 답변하는 RAG(Retrieval-Augmented Generation) 기반 챗봇.

- 문서를 청킹·벡터화하여 의미 기반 검색
- 답변마다 근거 문서와 위치를 함께 제공해 신뢰도 확보
- Slack·GitHub 연동으로 기존 업무 흐름에 자연스럽게 통합

| **실무자** (일반 사용자) | 빠르고 정확한 답변 + 출처 확인     | 웹 UI 또는 Slack에서 질문       |

## 핵심 기능

### 1. 문서 업로드

- PDF, MD, TXT — Drag & Drop 또는 파일 선택
- 업로드 즉시 청킹 → 벡터 임베딩 → ChromaDB 저장
- 파일 목록 UI: 인덱싱 중 / 준비 완료 / 오류 상태 표시
- 파일 삭제 및 재업로드

### 2. AI 채팅

- 질문 입력 → 관련 청크 검색 → OpenAI로 답변 생성
- SSE(Server-Sent Events) 스트리밍으로 실시간 타이핑 효과
- 마크다운 + 코드 블록 렌더링, 대화 기록 유지

### 3. 출처 표시

- 답변 하단에 출처 뱃지 (파일명 + 페이지/섹션)
- 뱃지 클릭 시 원문 하이라이트 사이드패널 오픈
- 신뢰 기반 답변으로 문서 검증 가능

### 4. GitHub 연동

- 레포 URL 입력 → README, 코드 주석 자동 크롤링 → 인덱싱
- 인덱싱 진행률 실시간 표시 (SSE)
- 코드 블록 Syntax Highlighting

### 5. Slack 연동

- `@techdocs [질문]` 멘션 → RAG 답변을 채널에 전송
- 출처 링크 포함, 기존 워크플로우 유지

### 6. 문서 비교 (선택)

- 특정 문서만 검색 범위 지정
- v1 vs v2 나란히 보기 (diff 뷰)

## 기술 결정 근거

| 선택                   | 이유                                                                                                       |
| ---------------------- | ---------------------------------------------------------------------------------------------------------- |
| Next.js 14 App Router  | 서버 컴포넌트로 초기 로딩 최적화, SSE 처리 용이                                                            |
| FastAPI                | Python 생태계(LangChain, OpenAI SDK) 활용, 비동기 스트리밍 지원                                            |
| ChromaDB               | 로컬 실행 가능, 설정 없이 빠른 프로토타이핑                                                                |
| Vercel + Railway       | 프론트/백엔드 분리 배포, 각 플랫폼 무료 티어 활용                                                          |
| JIRA + Claude Code MCP | 개발 태스크를 JIRA 이슈로 관리하고 Claude Code에서 MCP로 직접 조회·업데이트 — AI 보조 개발 워크플로우 구성 |

## 개발 워크플로우

### JIRA + Claude Code MCP

전체 개발 태스크는 JIRA 프로젝트(key: `SCRUM`)에서 관리한다.

- **Epic** 단위: PLAN.md의 Day별 작업 묶음 (Day 4~14)
- **Task** 단위: 각 Day 내 세부 구현 항목
- Claude Code의 JIRA MCP 서버를 통해 이슈 조회·상태 업데이트를 대화 중에 직접 수행
