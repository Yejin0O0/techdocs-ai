# PRD — Source Display Backend

## 1. 개요

AI 답변 생성 시 참고한 문서 청크의 출처 정보(파일명, 페이지 번호, 청크 원문)를 함께 반환하여 프론트엔드의 출처 표시 기능을 지원한다.

---

## 2. 사용자 스토리

| ID    | 역할   | 스토리                                  | 완료 조건                                      |
| ----- | ------ | --------------------------------------- | ---------------------------------------------- |
| US-01 | 실무자 | 답변이 어떤 문서를 참고했는지 알고 싶다 | sources 이벤트로 파일명이 반환된다             |
| US-02 | 실무자 | 출처의 정확한 위치(페이지)를 알고 싶다  | PDF 청크는 페이지 번호가 포함되어 반환된다     |
| US-03 | 실무자 | 출처 원문을 확인하고 싶다               | 청크 원문 텍스트가 sources에 포함되어 반환된다 |

---

## 3. 기술 결정 (ADR)

### ADR-01: sources 응답 형식 변경

**Context** — 기존 sources는 `string[]`(파일명 배열)이었으나, 프론트의 사이드패널 기능을 위해 페이지 번호와 청크 원문도 함께 반환해야 한다.

**Decision** — sources 형식을 `{ fileName, page?, chunk }[]`로 변경한다. 최대 3개, 유사도 높은 순으로 반환한다.

**Alternatives**

- _별도 엔드포인트_ — `/chat`과 `/sources` 두 번 호출하는 방식. 순서 보장이 어렵고 클라이언트 복잡도 증가
- _전체 문서 반환_ — 청크 대신 문서 전체를 반환하면 원문 검색이 정확하지만 페이로드가 너무 커짐

**Consequences**

- (+) 기존 SSE 구조를 그대로 활용, 추가 엔드포인트 불필요
- (+) 하이브리드 검색 점수 순서와 일치한 출처 순서 보장
- (-) 기존 `sources: string[]` 타입을 쓰는 프론트 코드 수정 필요
- (-) 청크 원문이 포함되어 SSE 페이로드 크기 증가

---

### ADR-02: PDF 페이지 번호 추출 방식

**Context** — 청킹 시 각 청크가 몇 번째 페이지에서 나왔는지 알아야 출처에 페이지 번호를 포함할 수 있다. 현재 pdfplumber로 전체 텍스트를 추출한 뒤 LangChain으로 청킹하는 구조라, 청킹 후 페이지 정보가 사라진다.

**Decision** — PDF 페이지별로 텍스트를 추출하고 청킹 시 페이지 번호를 metadata에 함께 저장한다.

**Alternatives**

- _전체 텍스트 추출 후 청킹 (현재 방식)_ — 구현 단순하나 청크가 어느 페이지에서 나왔는지 알 수 없음
- _청크 오프셋으로 역산_ — 전체 텍스트에서 청크 위치를 찾아 페이지를 역산. 구현 복잡하고 정확도 불확실

**Consequences**

- (+) 페이지 단위로 추출하므로 페이지 번호 정확히 보존
- (+) MD·TXT는 기존 로직 유지, PDF만 변경
- (-) 페이지 경계에서 잘린 문장의 청크는 앞 페이지 번호로 귀속
- (-) 업로드 로직 수정 후 기존 문서 재인덱싱 필요

---

## 4. Out of Scope

- 출처 3개 초과 반환
- 전체 문서 내용 반환 (청크만 반환)
- 출처 캐싱 또는 서버 저장
- 청크 내 정확한 문자 위치(offset) 반환

---

## 5. 용어 정의

| 용어    | 정의                                                                |
| ------- | ------------------------------------------------------------------- |
| 청크    | 문서를 검색에 적합한 크기(1000토큰/200overlap)로 잘라낸 텍스트 단위 |
| page    | PDF에서 청크가 추출된 페이지 번호. MD·TXT는 해당 없음               |
| sources | /chat SSE 응답의 마지막 이벤트. 참고한 청크의 출처 정보 배열        |
