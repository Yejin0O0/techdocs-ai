# UX 디테일 Frontend — Spec Fixed

## 기능 개요

로딩, 에러, 빈 상태, 다크모드, 반응형 등 전반적인 UX 상태를 처리한다.

---

## Skeleton 로딩

### 채팅 말풍선

- 전송 버튼 클릭 후 첫 토큰 수신 전까지 Skeleton UI 표시
- 첫 토큰 수신 시 Skeleton → 텍스트로 즉시 교체
- 기존 스트리밍 커서 애니메이션(`animate-pulse`)은 첫 토큰 이후 유지

### 파일 인덱싱

- 인덱싱 중(`indexing`) 상태일 때 파일 목록 아이템에 Skeleton 표시
- 기존 상태 뱃지와 공존 (Skeleton은 파일명/크기 영역)

---

## 에러 상태

### 채팅 API 실패

- 에러 메시지를 assistant 말풍선 위치에 표시
- 말풍선 하단에 **재시도** 버튼 표시
- 재시도 클릭 시 마지막 질문을 자동으로 다시 전송

### 파일 업로드 실패

- 기존 `error` 상태 뱃지 + 에러 메시지 유지 (이미 구현됨)

---

## 빈 상태

### 문서 없음

- 파일 목록이 비어있을 때 안내 문구 + 업로드 유도 UI 표시
- 예: "문서를 업로드하면 AI가 답변할 수 있어요"

### 대화 없음

- 채팅 기록이 없을 때 질문 예시 3개 표시
- 예시 클릭 시 입력창에 자동 채워짐 (사용자가 수정 후 전송 가능)
- 질문 예시는 업로드된 문서 유무와 무관하게 항상 표시

---

## 다크모드

- 시스템 설정에 따라 자동 적용 (`prefers-color-scheme`)
- 헤더 우측에 토글 버튼 배치 (Sun/Moon 아이콘)
- 수동 전환 시 선택값을 `localStorage`에 저장, 새로고침 후에도 유지
- 구현: Tailwind `dark:` 클래스 + `next-themes` 라이브러리

---

## 반응형

- Tailwind 브레이크포인트 기준: `sm`(640px), `md`(768px), `lg`(1024px)
- **모바일**: 업로드 패널과 채팅 패널을 탭으로 전환
- **태블릿 이상**: 업로드 패널(좌) + 채팅 패널(우) 나란히 배치 (현재 데스크탑 레이아웃)
- SourcePanel은 모바일에서 전체 너비(`w-full`)로 표시

---

## Out of Scope

- 서버 사이드 다크모드 (hydration mismatch 방지를 위해 클라이언트 전용)
- 에러 로그 수집 / Sentry 연동
- 업로드 에러 자동 재시도
