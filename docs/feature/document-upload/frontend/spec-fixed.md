# Document Upload — Spec Fixed

## 기능 개요

PDF, MD, TXT 파일을 드래그앤드롭 또는 클릭으로 업로드하면 자동으로 인덱싱되어 AI 채팅에서 질문할 수 있는 상태로 준비되는 기능.

---

## 핵심 동작

- 파일 선택 즉시 업로드 시작 (별도 확인 버튼 없음)
- PDF, MD, TXT 형식만 허용 / 최대 10MB
- 다중 파일 동시 업로드 가능 / 문서 수 제한 없음
- 업로드 후 상태 흐름: `업로드 중` → `인덱싱 중` → `준비 완료`
- 오류 발생 시 `오류` 상태와 함께 원인 메시지 + 재시도 버튼 표시
- 지원하지 않는 형식 또는 크기 초과 시 드롭존 하단에 에러 텍스트 표시
- `uploading` 상태는 프론트 로컬에서만 관리, `indexing` / `ready` / `error`는 백엔드 응답 기준
  - `uploading`은 브라우저에서 서버로 파일이 전송되는 순간의 상태로, 다른 사용자와 공유할 필요가 없음
  - `indexing` 이후는 서버 처리 상태이므로 모든 사용자가 동일하게 알아야 함 (공유 문서 풀 정책)
- 목록은 최신순 정렬 (새로 업로드한 파일이 맨 위)
- 목록이 길어지면 목록 영역만 스크롤, 드롭존은 상단에 고정
- 처리 중인 파일이 있어도 드롭존은 계속 활성화 (추가 업로드 가능)

---

## 페이지 로드 동작

- 페이지 진입 시 백엔드 `GET /docs` 호출로 기존 문서 목록을 불러와 `documents` state에 세팅
- `uploading` 상태 파일이 전송 중 새로고침되면 로컬 state는 사라지지만, 서버는 처리 결과(`indexing` / `error`)를 반환하므로 `GET /docs` 응답으로 자연스럽게 반영됨

---

## 데이터 구조 참고

- 문서 풀은 모든 사용자가 공유 (로그인 없음)
- `uploadedBy` 필드를 optional로 보유 — 추후 로그인 기능 추가 시 사용자별 문서 필터링에 활용
- `file` 필드를 optional로 보유 — 오류 재시도 시 File 객체를 다시 전송하기 위해 인메모리에 보관. 백엔드 연결 후 서버에서 재처리 가능해지면 제거 고려

---

## 삭제 기능

- 각 항목에 삭제 버튼 표시, hover 시에만 노출
- 모든 상태(`uploading`, `indexing`, `ready`, `error`)에서 삭제 가능
- 확인 다이얼로그 없이 즉시 삭제
- 단일 항목 삭제만 지원 (다중 선택 없음)

---

## 엣지 케이스

- **중복 파일명** — 업로드 전 `documents` 상태에서 같은 파일명 감지 시 "이미 같은 이름의 문서가 있어요. 덮어쓸까요?" 경고 표시. 확인 시 업로드 진행, 취소 시 업로드 안 함. 새 파일은 즉시 업로드 흐름 유지
- **0바이트 파일** — 프론트에서 감지 후 업로드 차단, 드롭존 하단에 에러 텍스트 표시 (4초 후 자동 사라짐)
- **오류 상태 재시도** — 오류 항목에 재시도 버튼 표시, 클릭 시 동일 파일 재업로드 시작
